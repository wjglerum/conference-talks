# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

See [AGENTS.md](AGENTS.md) for the generic Quarkus workflow rules (extension-first, Dev MCP tooling, testing via subagents). The notes below cover what is specific to this project.

## What this is

The source for [wjglerum.nl](https://wjglerum.nl/), a personal list of conference talks, podcasts and workshops. It is a static site generated by [Quarkus Roq](https://iamroq.dev/) and deployed to GitHub Pages. There is no runtime backend: the build produces flat HTML under `target/roq/`.

## Commands

- `./mvnw quarkus:dev` then open http://localhost:8080/. Hot reload picks up changes to talks, templates, data and styles.
- `./mvnw test` runs the JUnit tests (`@QuarkusTest` + REST Assured).
- The static site is generated in CI by `quarkiverse/quarkus-roq@v1`, not by a local Maven goal. To preview locally, use dev mode.
- Java 25 is required (`maven.compiler.release` is 25, CI uses `java-version: 25`).

## Architecture

Content is data, not code. Each talk is one markdown file in `src/main/resources/content/talks/` named `YYYY-MM-DD-slug.md` with YAML front matter. The `site.collections.talks` config in `application.properties` turns that directory into a Roq collection; `talks.future=true` (and CI's `site-future: 'true'`) means dated-in-the-future talks are published.

Rendering is template-driven via Qute:
- `templates/layouts/main.html` renders the landing page (`content/index.html`), which iterates `site.collections.talks` into a card grid plus a Leaflet map.
- `templates/layouts/talk.html` renders each talk detail page. A talk's `layout: talk` front matter selects it.
- `templates/partials/` holds the shared includes: `seo.html` (the `<title>` plus og/twitter meta, taking an `ogType` parameter, used by both layouts), `theme-head.html`, `theme-toggle.html` and `tour-block.html`. Reference them with the `partials/` prefix, e.g. `{#include partials/seo ogType='website' /}`.
- Talk `type` (`talk | meetup | deepdive | workshop | podcast | techtalk`) drives the colored badge in both templates. When adding a new type, update the `{#if page.data("type") == ...}` chains in BOTH templates and add styling in `public/site.css`.
- `site.title` and `site.description` come from the `content/index.html` front matter (the Roq convention), not from `application.properties`. `seo.html` only appends `site.title` to the page title when they differ, so the landing page is not titled "On Tour · On Tour".

The sitemap is generated by the `quarkus-roq-plugin-sitemap` extension. It needs a content page (`content/sitemap.xml`) that does `{#include fm/sitemap.xml /}`; the dependency alone does not register the route. Pages opt out with `sitemap: false` front matter. `public/robots.txt` points crawlers at it. Plugin versions track core Roq, so both are pinned via the `${quarkus-roq.version}` property in `pom.xml`.

The map ties content to geography. `data/cities.yml` maps `"City, Country"` strings to `{ lat, lng }`. `Cities.java` is a Roq `@DataMapping("cities")` record that exposes this file as a CDI bean. `index.html` looks up each talk's `location` via `cdi:cities.list.get(...)` to emit JSON coordinates for the markers. A talk whose `location` is not in `cities.yml` silently drops off the map, so add new locations there.

## Qute gotchas in index.html

The map's `<script>` block is wrapped in a Qute raw block `{| ... |}` so the JavaScript `{ }` are not parsed as Qute expressions. The inline JSON uses escaped braces `\{ ... \}` and `{#let}`/`{#if}` to emit per-talk coordinates. Edit this region carefully: an unescaped brace breaks the whole page build.

Do not use the elvis operator inside `{#let}`: `{#let x=a ?: b}` splits on the space and fails to parse. Use elvis in a plain expression instead (`{a ?: b}`), as `partials/seo.html` does for the description fallback.
